Create `tests/ui/test_frontend.py` with 6 browser tests for homepage, navbar, responsive layout, and touch targets.

**Imports:** pytest, re (for text matching), from playwright.sync_api import expect

**Tests:**

`test_homepage_hero_and_ctas` (TFEND-01):
- `page.goto(base_url)`
- `expect(page.get_by_text("Ready to bring your community together?")).to_be_visible()`
- Verify auth-aware CTA (logged-out state — default for fresh page):
  - `expect(page.get_by_text("Get Started")).to_be_visible()` — the logged-out CTA
- Verify feature section exists: at least look for a heading or section below the hero. Based on research, feature cards are present. Look for common feature text or verify the page has multiple sections.

`test_navbar_logged_out` (TFEND-02):
- `page.goto(base_url)`
- Set viewport to 1280x800 (desktop) to ensure desktop nav is visible
- `expect(page.get_by_text("Log In")).to_be_visible()`
- `expect(page.get_by_text("Sign Up")).to_be_visible()`
- Note: Logged-in navbar state (Dashboard, etc.) requires auth token injection — deferred to Plan 05

`test_mobile_hamburger_menu` (TFEND-03):
- `page.set_viewport_size({"width": 375, "height": 812})`
- `page.goto(base_url)`
- hamburger = `page.get_by_role("button", name="Toggle menu")`
- `expect(hamburger).to_be_visible()`
- `hamburger.click()`
- After click, verify mobile menu panel has nav links:
  - `expect(page.get_by_text("Log In")).to_be_visible()` (in mobile menu)
  - `expect(page.get_by_text("Sign Up")).to_be_visible()` (in mobile menu)

`test_responsive_no_overflow` (TFEND-07 + TFEND-09 combined):
- Parametrize with `@pytest.mark.parametrize("width,height", [(375, 812), (768, 1024), (1280, 800)])`
- `page.set_viewport_size({"width": width, "height": height})`
- `page.goto(base_url)`
- Wait for page to settle: `page.wait_for_load_state("networkidle")`
- Check no horizontal overflow: `overflow = page.evaluate("document.documentElement.scrollWidth > window.innerWidth")`
- `assert not overflow, f"Horizontal overflow detected at {width}px viewport"`
- This covers both TFEND-07 (responsive at all three widths) and TFEND-09 (no horizontal overflow)

`test_touch_targets_mobile` (TFEND-08):
- `page.set_viewport_size({"width": 375, "height": 812})`
- `page.goto(base_url)`
- Test key interactive elements for minimum 44x44px:
  - hamburger_btn = `page.get_by_role("button", name="Toggle menu")`
  - `expect(hamburger_btn).to_be_visible()`
  - box = `hamburger_btn.bounding_box()`
  - `assert box is not None, "Hamburger button not found"`
  - `assert box["width"] >= 44, f"Hamburger width {box['width']}px < 44px minimum"`
  - `assert box["height"] >= 44, f"Hamburger height {box['height']}px < 44px minimum"`
- Also test navigation links (after opening hamburger menu):
  - `hamburger_btn.click()`
  - For at least one visible nav link (e.g., "Log In"):
    - link = `page.get_by_text("Log In").first`
    - `expect(link).to_be_visible()`
    - link_box = `link.bounding_box()`
    - `assert link_box is not None`
    - `assert link_box["height"] >= 44, f"Nav link height {link_box['height']}px < 44px"`

**Key patterns:**
- All tests use the `page` fixture from pytest-playwright (auto-provided)
- All tests use `base_url` fixture from tests/ui/conftest.py
- Use `from playwright.sync_api import expect` for Playwright assertions
- No `time.sleep()` — use Playwright's built-in auto-waiting
- Each test has `@pytest.mark.req("TFEND-XX")` marker
- Tests that need specific viewports call `page.set_viewport_size()` BEFORE `page.goto()`
