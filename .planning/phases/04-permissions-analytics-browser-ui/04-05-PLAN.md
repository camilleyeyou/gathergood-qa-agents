1. `login_via_ui(page, base_url, email, password)` — module-level helper function (not a fixture):
   - `page.goto(f"{base_url}/login")`
   - Fill email field: `page.get_by_label("Email").fill(email)` or `page.locator("input[type='email']").fill(email)`
   - Fill password field: `page.get_by_label("Password").fill(password)` or `page.locator("input[type='password']").fill(password)`
   - Click submit: `page.get_by_role("button", name="Log In").click()` or `page.locator("button[type='submit']").click()`
   - Wait for navigation: `page.wait_for_url(f"{base_url}/**", timeout=10000)` — wait until URL changes from /login
   - Note: The exact selectors depend on the live frontend. Try label-based first, fall back to type-based. The research confirms Log In button text.

2. `ui_test_user` fixture (module-scoped):
   - Register a fresh user via httpx: POST `/auth/register/` with unique_email(), password "TestPass123!"
   - Login to get tokens (for API data setup): POST `/auth/login/`
   - Return dict: `{"email": email, "password": "TestPass123!", "access": tokens["access"], "first_name": "UI", "last_name": "Tester"}`
   - Note: This user is separate from the session auth_client user, avoiding interference

3. `ui_auth_client` fixture (module-scoped):
   - Creates an httpx.Client with the ui_test_user's JWT for API data setup
   - `httpx.Client(base_url=_settings.API_URL, headers={"Authorization": f"Bearer {ui_test_user['access']}"}, timeout=30)`
   - Yields client, closes on teardown

4. `ui_checkout_data` fixture (module-scoped):
   - Uses `ui_auth_client` to create org, event, free tier, and publish event
   - Steps:
     a. Create org: POST `/organizations/` with `{"name": org_name(), "description": "UI test org"}`
     b. Create event: POST `/organizations/{org_slug}/events/` with standard event JSON
     c. Create free tier: POST `.../ticket-tiers/` with `{"name": tier_name(), "price": "0.00", "quantity_total": 100, "min_per_order": 1, "max_per_order": 10, "visibility": "PUBLIC"}`
     d. Publish: POST `.../publish/`
   - Return dict: `{"org_slug": org_slug, "event_slug": event_slug, "tier_id": tier_id, "event_title": event_title}`
   - Note: No teardown_registry dependency — UI tests create their own data. The org and event use RUN_ID-prefixed names for isolation.
