**Imports:** httpx, pytest, event_title/tier_name from factories.common, assert_status from helpers.api, Settings from settings

**Module-level:** `_settings = Settings()`

**Module-scoped fixture — `visibility_event`:**
TPUBL-04/05 need an event with PUBLIC, HIDDEN, and INVITE_ONLY tiers to verify filtering. Create:
1. Create event via `auth_client.post(f"/organizations/{org['slug']}/events/", json={...})`
2. Register in teardown_registry
3. Create 3 tiers:
   - PUBLIC tier: `{"name": f"Public-{tier_name()}", "price": "0.00", "quantity_total": 10, "min_per_order": 1, "max_per_order": 5, "visibility": "PUBLIC"}`
   - HIDDEN tier: `{"name": f"Hidden-{tier_name()}", "price": "0.00", "quantity_total": 10, "min_per_order": 1, "max_per_order": 5, "visibility": "HIDDEN"}`
   - INVITE_ONLY tier: `{"name": f"Invite-{tier_name()}", "price": "0.00", "quantity_total": 10, "min_per_order": 1, "max_per_order": 5, "visibility": "INVITE_ONLY"}`
4. Register all tiers in teardown_registry
5. Publish the event
6. Return dict: `{"event": pub_event, "public_tier": public_tier, "hidden_tier": hidden_tier, "invite_tier": invite_tier}`

**Tests:**

`test_public_browse_events` (TPUBL-01):
- Use unauthenticated `httpx.get(_settings.API_URL + "/public/events/")` (no auth header)
- Assert status 200
- Assert response is a list
- Test `?category=MEETUP` filter: `httpx.get(_settings.API_URL + "/public/events/?category=MEETUP")` — assert 200, assert isinstance list
- Test `?q=` search filter with part of a known event title: `httpx.get(_settings.API_URL + "/public/events/?q=test")` — assert 200, assert isinstance list
- Add comment: `# DEVIATION: ?format= filter returns 404 — not supported by live API. Skipped.`

`test_public_browse_excludes_draft` (TPUBL-02):
- Use unauthenticated `httpx.get(_settings.API_URL + "/public/events/", timeout=30)`
- Assert status 200
- Iterate all events in response: for each event, assert `ev.get("status") not in ("DRAFT", "CANCELLED")`, with message showing the offending event
- Note: we can't assert our DRAFT `event` fixture is absent by title (other test runs may have data), but we verify the invariant that NO draft/cancelled events appear

`test_public_org_page` (TPUBL-03):
- Use unauthenticated `httpx.get(_settings.API_URL + f"/public/{org['slug']}/", timeout=30)`
- Assert status 200
- Assert `"organization"` in data
- Assert `"events"` in data
- Assert `isinstance(data["events"], list)`
- Assert the org name or slug matches: `data["organization"]["slug"] == org["slug"]`

`test_public_event_detail_public_tiers_only` (TPUBL-04):
- Use unauthenticated `httpx.get(_settings.API_URL + f"/public/{org['slug']}/events/{visibility_event['event']['slug']}/", timeout=30)`
- Assert status 200
- Assert `"ticket_tiers"` in data
- tier_names = `[t["name"] for t in data["ticket_tiers"]]`
- Assert at least one tier name starts with `"Public-"` (the PUBLIC tier we created)
- TPUBL-05 check (combined): Assert NO tier name starts with `"Hidden-"` or `"Invite-"`
- Add comment explaining: `# ticket_tiers in public response do NOT include "visibility" field — it's stripped. We verify by tier name prefixes.`

`test_public_event_hides_non_public_tiers` (TPUBL-05):
- This is a dedicated test for the negative case. Use same unauthenticated GET as TPUBL-04
- Assert status 200
- tier_names = `[t["name"] for t in data["ticket_tiers"]]`
- Assert `not any(n.startswith("Hidden-") for n in tier_names)`, message: "HIDDEN tier should not appear in public detail"
- Assert `not any(n.startswith("Invite-") for n in tier_names)`, message: "INVITE_ONLY tier should not appear in public detail"
- Assert `len(data["ticket_tiers"]) == 1`, message: "Only the PUBLIC tier should appear"

**Key patterns:**
- Public endpoints use raw `httpx.get()` with `timeout=30` — NOT `auth_client` (these are unauthenticated)
- Each test has `@pytest.mark.req("TPUBL-XX")` marker
- `visibility_event` fixture is module-scoped and creates its own event for isolation
