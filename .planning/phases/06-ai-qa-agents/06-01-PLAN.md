     - `type_text(text: str)`: calls `self.page.keyboard.type(text)`
     - `key(key_combo: str)`: calls `self.page.keyboard.press(key_combo)`
     - `scroll(x: int, y: int, direction: str, amount: int)`: moves mouse to (x,y), then calls `self.page.mouse.wheel(delta_x, delta_y)` where delta = amount * 100, direction maps "down" to positive delta_y, "up" to negative delta_y, "right" to positive delta_x, "left" to negative delta_x
     - `mouse_move(x: int, y: int)`: calls `self.page.mouse.move(x, y)`
     - `execute_action(action: str, params: dict) -> str`: dispatches to the above methods based on `action` string. For "screenshot" returns the b64 string directly. For all other actions, executes the action then calls `self.screenshot()` and returns the result. Wraps the dispatch in try/except; on PlaywrightError, returns a screenshot anyway (so Claude sees the error state).
   - Import `base64` from stdlib and `Page` from `playwright.sync_api`.

   - Function signature: `run_agent_scenario(backend: PlaywrightComputerBackend, client: anthropic.Anthropic, goal: str, system_prompt: str | None = None, max_iterations: int = MAX_ITERATIONS, max_input_tokens: int = MAX_INPUT_TOKENS) -> dict`
   - Implementation follows RESEARCH.md Pattern 2 exactly:
     a. Take initial screenshot via `backend.screenshot()`
     b. Build initial messages list with image (base64 PNG) + text (goal)
     c. Loop up to `max_iterations`:
        - **Cost cap check (per AIQA-10):** Before each API call, check if `total_input_tokens >= max_input_tokens`. If exceeded, break the loop immediately and set verdict to "INCONCLUSIVE" with reasoning "Cost cap reached: {total_input_tokens} input tokens exceeded limit of {max_input_tokens}".

        - Iterate response.content blocks: collect text into `final_text`, for tool_use blocks with name=="computer" call `backend.execute_action(block.input["action"], block.input)` and build tool_result with screenshot image
        - If no tool_results, break (Claude is done)
        - Append tool_results as user message
        - Track `input_tokens` and `output_tokens` from `response.usage`
     d. Parse verdict from final_text: search for "PASS", "FAIL", "INCONCLUSIVE" (case-insensitive). Default to "INCONCLUSIVE".
     e. Return dict: `{"verdict": str, "reasoning": final_text, "steps": iterations, "input_tokens": total_input, "output_tokens": total_output}`
