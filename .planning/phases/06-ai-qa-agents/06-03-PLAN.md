2. Update `tests/ai_agents/conftest.py` to add the following (append to existing content, do NOT replace existing fixtures):

   a. A `pytest_runtest_makereport` hook (using `@pytest.hookimpl(hookwrapper=True)`) that:
      - Yields to get the report
      - Checks if `report.when == "call"` and the test item has an `ai_agent` marker
      - Looks for `_agent_result` attribute on the test item (set by a wrapper, see below)
      - If found, creates an HTML extras section with:
        - Verdict (PASS/FAIL/INCONCLUSIVE) in bold
        - Steps count and token usage (input_tokens, output_tokens)
        - Full reasoning text in a `<pre>` block
      - Attaches it via `report.extras` (using `pytest_html.extras.html(...)`)
      - Also prints the agent summary to stdout for console visibility:
        ```
        [AI Agent] Verdict: {verdict} | Steps: {steps} | Tokens: {input_tokens} in / {output_tokens} out
        [AI Agent] Reasoning: {reasoning[:200]}...
        ```

   b. A `record_agent_result` fixture (scope="function") that:
      - Returns a callable `_record(result: dict)`
      - When called, stores `result` on `request.node._agent_result = result`
      - This is the bridge between test functions and the report hook

   c. Update the existing `agent_system_prompt` fixture or add an `agent_config` fixture (scope="session") that returns a dict:
      ```python
      {"max_iterations": 20, "model": "claude-sonnet-4-6"}
      ```
      This makes the configuration centralized and overridable.

3. Document in each scenario test file (Plan 02's files) that they should call `record_agent_result(result)` after `run_agent_scenario`. Since Plan 02 may execute in parallel with this plan, add a NOTE comment in conftest.py explaining the pattern:

```python
# NOTE: Test functions should call record_agent_result(result) after run_agent_scenario

```

4. Also add `record_agent_result` calls to ALL scenario test files from Plan 02. Read each file in tests/ai_agents/scenarios/ and update each test function to:

   - Call `record_agent_result(result)` immediately after the `run_agent_scenario(...)` call, before the assert

The `pytest_runtest_makereport` hook should gracefully handle the case where `_agent_result` is not set (test crashed before recording, or test doesn't use the fixture) — just skip the extras attachment.
