Update `tests/ui/conftest.py` to enable Playwright tracing for all browser tests, so that failed tests produce trace files that conftest_report.py can attach to the HTML report (TRPT-03).

**Add a pytest fixture that wraps browser context with tracing:**
- Add a `context` fixture (scope="function") that overrides the default pytest-playwright context
- In the fixture setup: `context.tracing.start(screenshots=True, snapshots=True, sources=True)`
- Yield the context
- In teardown: `context.tracing.stop(path=trace_path)` where trace_path is `reports/traces/{request.node.nodeid_sanitized}.zip`
- Sanitize nodeid by replacing `/`, `::`, and other path-unsafe chars with `_`
- Only save the trace if the test failed (check via `request.node.rep_call.failed` -- this requires the `pytest_runtest_makereport` hook to stash the report on the item, which conftest_report.py already does)
- Alternative simpler approach: always save traces (they are small zip files), and conftest_report.py only attaches them for failed tests. This avoids the complexity of checking failure status in the fixture.

**Simpler approach (preferred):** Always start and stop tracing, always save the trace zip. conftest_report.py already only attaches traces for failed tests. This keeps the fixture simple.

Implementation:
```python
import os

@pytest.fixture(scope="function")
def context(browser, request):
    """Browser context with Playwright tracing enabled for all UI tests."""
    ctx = browser.new_context()
    ctx.tracing.start(screenshots=True, snapshots=True)
    yield ctx
    # Save trace
    trace_dir = os.path.join("reports", "traces")
    os.makedirs(trace_dir, exist_ok=True)
    safe_name = request.node.nodeid.replace("/", "_").replace("::", "_").replace("[", "_").replace("]", "_")
    trace_path = os.path.join(trace_dir, f"{safe_name}.zip")
    ctx.tracing.stop(path=trace_path)
    ctx.close()
```

Also add a `page` fixture that creates a page from this traced context:
```python
@pytest.fixture(scope="function")
def page(context):
    """Page from traced browser context."""
    pg = context.new_page()
    yield pg
    pg.close()
```

Note: This overrides pytest-playwright's default `page` and `context` fixtures. The `browser` fixture comes from pytest-playwright and is still used.
