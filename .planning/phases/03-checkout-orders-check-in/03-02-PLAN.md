**TORDR-01** `test_list_own_orders(auth_client, completed_order)`:
- GET /orders/
- assert_status(resp, 200)
- orders = resp.json(); assert isinstance(orders, list); assert len(orders) >= 1
- Find our order: any(o["id"] == completed_order["id"] for o in orders)

**TORDR-02** `test_view_order_detail(auth_client, completed_order)`:
- GET /orders/{completed_order["id"]}/
- assert_status(resp, 200)
- data = resp.json(); assert data["id"] == completed_order["id"]
- assert "confirmation_code" in data; assert "tickets" in data; assert "line_items" in data

**TORDR-03** `test_lookup_by_confirmation_code_no_auth(completed_order)`:
- Create a plain httpx.Client (no auth) with base_url=Settings().API_URL, timeout=30
- GET /orders/lookup/{completed_order["confirmation_code"]}/
- assert_status(resp, 200)
- data = resp.json(); assert data["confirmation_code"] == completed_order["confirmation_code"]
- Also test 404 for bogus code: GET /orders/lookup/ZZZZZZZZZZ/ -> assert resp.status_code == 404
- Close the plain client in a try/finally

**TORDR-04** `test_confirmation_code_format(completed_order)`:
- code = completed_order["confirmation_code"]
- assert re.match(r'^[A-Z0-9]{10}$', code), f"Code format invalid: {code}"
- NOT [A-Za-z0-9] — uppercase only (Pitfall 7)

**TORDR-05** `test_list_own_tickets(auth_client, completed_order)`:
- GET /tickets/
- assert_status(resp, 200)
- tickets = resp.json(); assert isinstance(tickets, list); assert len(tickets) >= 1
- Find our ticket: any(t["id"] == completed_order["tickets"][0]["id"] for t in tickets)
- Verify qr_code_data present: ticket = next(t for t in tickets if t["id"] == completed_order["tickets"][0]["id"]); assert ticket["qr_code_data"]

**TORDR-06** `test_qr_code_data_format(completed_order)`:
- qr = completed_order["tickets"][0]["qr_code_data"]
- UUID = r'[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}'
- QR_RE = f'^{UUID}:{UUID}:{UUID}:[0-9a-f]{{16}}$'
- assert re.match(QR_RE, qr), f"QR format invalid: {qr}"
- parts = qr.split(":"); Note: splitting on ":" produces 7 parts (3 UUIDs have dashes, not colons, so split on ":" gives: first UUID, second UUID segments... Actually UUIDs contain dashes not colons.
- Better: split by regex. Use groups: m = re.match(f'^({UUID}):({UUID}):({UUID}):([0-9a-f]{{16}})$', qr); assert m; assert len(m.group(4)) == 16

**TORDR-07** `test_qr_hmac_structure(completed_order)`:
- qr = completed_order["tickets"][0]["qr_code_data"]
- Extract last segment after the last colon: hmac_part = qr.split(":")[-1] — WAIT, UUIDs have dashes not colons. The colon delimiter separates the 3 UUIDs and the HMAC.
- Actually the format is order_uuid:tier_uuid:ticket_uuid:hmac. Count colons: UUID format is xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx (no colons). So qr.split(":") gives exactly 4 parts.
- parts = qr.split(":"); assert len(parts) == 4, f"Expected 4 colon-separated parts: {qr}"
- hmac_seg = parts[3]; assert re.match(r'^[0-9a-f]{16}$', hmac_seg), f"HMAC not 16 hex: {hmac_seg}"
- Also verify first 3 parts are UUIDs: for p in parts[:3]: assert re.match(UUID, p)

Each test decorated with @pytest.mark.req("TORDR-NN").
