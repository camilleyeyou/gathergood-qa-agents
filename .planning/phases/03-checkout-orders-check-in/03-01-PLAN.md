Create tests/api/test_checkout.py with the following 12 tests. ALL use `POST /checkout/` with action field — NEVER use /checkout/calculate/ or /checkout/free/ (those return 404).

Module docstring: "Checkout API tests — TCHKT-01 through TCHKT-12."

Import pytest, re, assert_status from helpers.api, event_title and tier_name from factories.common.

**TCHKT-01** `test_calculate_returns_pricing_breakdown(auth_client, org, checkout_event)`:
- POST /checkout/ with action="calculate", org_slug=org["slug"], event_slug=checkout_event["event"]["slug"], items=[{"tier_id": checkout_event["free_tier"]["id"], "quantity": 2}], promo_code=""
- assert_status(resp, 200)
- data = resp.json(); assert "line_items" in data; assert "subtotal" in data; assert "discount_amount" in data; assert "fees" in data; assert "total" in data; assert "is_free" in data
- assert data["is_free"] is True (free tier)
- assert len(data["line_items"]) == 1; assert data["line_items"][0]["tier_id"] == checkout_event["free_tier"]["id"]

**TCHKT-02** `test_free_checkout_completes_with_completed_status(auth_client, org, checkout_event)`:
- POST /checkout/ with action="complete", org_slug, event_slug, items=[{"tier_id": free_tier_id, "quantity": 1}], promo_code="", billing_name="Free Attendee", billing_email=unique_email(), billing_phone=""
- assert_status(resp, 201)
- data = resp.json(); assert data["status"] == "COMPLETED"

**TCHKT-03** `test_free_checkout_returns_confirmation_and_qr(auth_client, org, checkout_event)`:
- POST /checkout/ same as TCHKT-02 but with a different unique_email
- assert_status(resp, 201)
- data = resp.json(); assert re.match(r'^[A-Z0-9]{10}$', data["confirmation_code"])
- assert len(data["tickets"]) >= 1; qr = data["tickets"][0]["qr_code_data"]
- QR_RE = r'^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}:[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}:[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}:[0-9a-f]{16}$'
- assert re.match(QR_RE, qr), f"QR format invalid: {qr}"

**TCHKT-04** `test_paid_checkout_creates_payment_intent(auth_client, org, checkout_event)`:
- Decorated with @pytest.mark.skip(reason="Stripe not configured on live backend: POST /checkout/payment-intent/ returns 'You did not provide an API key.' No STRIPE_SECRET_KEY set.")
- pass (body is empty — skipped test)

**TCHKT-05** `test_checkout_requires_billing_fields(auth_client, org, checkout_event)`:
- Three sub-assertions:
  a) POST /checkout/ action="complete" WITHOUT billing_name (omit field entirely) -> assert_status(resp, 400); assert "billing_name" in resp.json()
  b) POST /checkout/ action="complete" WITHOUT billing_email -> assert_status(resp, 400); assert "billing_email" in resp.json()
  c) POST /checkout/ action="complete" WITH billing_name and billing_email but WITHOUT billing_phone -> assert_status(resp, 201) (phone is optional)
- All three use free_tier to avoid Stripe complications

**TCHKT-06** `test_checkout_rejects_capacity_exceeded(auth_client, org, checkout_event)`:
- MUST use action="calculate" (not "complete" — Pitfall 4)
- Create a tiny tier (quantity_total=2) on checkout_event, then calculate with quantity=5
- Helper: create tier via POST /organizations/{org['slug']}/events/{checkout_event['event']['slug']}/ticket-tiers/ with quantity_total=2, min_per_order=1, max_per_order=10
- POST /checkout/ action="calculate" with quantity=5 on that tier
- assert resp.status_code == 400; assert "remaining" in resp.json().get("detail", "").lower() or "tickets remaining" in resp.text.lower()

**TCHKT-07** `test_checkout_rejects_below_min_per_order(auth_client, org, checkout_event)`:

- Create a tier with min_per_order=3, max_per_order=10, quantity_total=50
- POST /checkout/ action="calculate" with quantity=1 on that tier
- assert resp.status_code == 400; assert "quantity must be between" in resp.json().get("detail", "").lower()

**TCHKT-08** `test_checkout_rejects_above_max_per_order(auth_client, org, checkout_event)`:
- MUST use action="calculate" (Pitfall 4)
- Use the same tier from TCHKT-07 approach: min_per_order=1, max_per_order=3, quantity_total=50

- assert resp.status_code == 400; assert "quantity must be between" in resp.json().get("detail", "").lower()

**TCHKT-09** `test_checkout_rejects_non_published_event(auth_client, org, checkout_event, teardown_registry)`:
- Create a DRAFT event (do NOT publish it) via _create_draft_event helper

- assert resp.status_code == 404 (NOT 400 — Pitfall 3)

**TCHKT-10** `test_expired_promo_returns_zero_discount(auth_client, org, checkout_event, expired_promo)`:
- POST /checkout/ action="calculate" with paid_tier, promo_code=expired_promo["code"]
- assert_status(resp, 200) (NOT 400 — Pitfall 2)

**TCHKT-11** `test_over_limit_promo_returns_zero_discount(auth_client, org, checkout_event, exhausted_promo)`:
- POST /checkout/ action="calculate" with paid_tier, promo_code=exhausted_promo["code"]
- assert_status(resp, 200) (NOT 400 — Pitfall 2)

**TCHKT-12** `test_promo_discount_applied_to_line_items(auth_client, org, checkout_event)`:
- POST /checkout/ action="calculate" with paid_tier (price $25), quantity=1, promo_code=checkout_event["promo"]["code"] (10% discount)
- assert_status(resp, 200)
- data = resp.json(); assert data["promo_applied"] == checkout_event["promo"]["code"]
