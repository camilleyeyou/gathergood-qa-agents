**TAUTH-01: test_register_user**
- Create a fresh `httpx.Client(base_url=settings.API_URL, timeout=30)` (NOT auth_client -- registration is unauthenticated)
- POST `/auth/register/` with:
  ```json
  {"email": unique_email(), "password": "TestPass123!", "password_confirm": "TestPass123!", "first_name": "Test", "last_name": "User"}
  ```
- `password_confirm` is REQUIRED (400 without it)
- assert_status(resp, 201, "POST /auth/register/")
- Assert `resp.json()["user"]["email"]` matches the email sent
- Assert `"id" in resp.json()["user"]`
- Assert `resp.json()["message"] == "Account created successfully."`
- Import Settings from settings module, instantiate to get API_URL

**TAUTH-02: test_login_returns_jwt_tokens**
- Register a user first (POST `/auth/register/`), then login
- POST `/auth/login/` with `{"email": email, "password": "TestPass123!"}`
- assert_status(resp, 200, "POST /auth/login/")
- Assert `"access" in data` and `"refresh" in data`
- Assert both tokens are non-empty strings with `len() > 10`

**TAUTH-03: test_token_refresh_issues_new_tokens**
- Register + login to get tokens
- POST `/auth/token/refresh/` with `{"refresh": login_data["refresh"]}`
- assert_status 200
- Assert response has BOTH `"access"` and `"refresh"` keys (rotation = new tokens issued)
- Assert `data["access"] != login_data["access"]` (new access token is different)
- Assert `data["refresh"] != login_data["refresh"]` (new refresh token is different)
- Do NOT assert that the old refresh token is rejected (per RESEARCH.md Pitfall 7: old tokens remain valid)

**TAUTH-04: test_password_reset_request**
- POST `/auth/forgot-password/` with `{"email": "nonexistent@example.com"}`
- assert_status 200
- Assert response contains a generic message (API returns "If an account exists..." regardless of email existence)
- This test verifies the endpoint responds, not that an email is delivered (manual-only per VALIDATION.md)

**TAUTH-05: test_password_reset_bad_token**
- POST `/auth/reset-password/` with `{"uid": "baduid", "token": "badtoken", "password": "NewPass123!", "password_confirm": "NewPass123!"}`
- assert_status(resp, 400, "POST /auth/reset-password/")
- Assert `"Invalid or expired reset link"` appears in response text

All tests that register users should use `unique_email()` from factories.common for isolation. Use a module-level fresh httpx.Client for unauthenticated calls (register, login, refresh, forgot-password, reset-password).

Create a module-level fixture or helper:
```python
import httpx
import pytest
from settings import Settings
from factories.common import unique_email
from helpers.api import assert_status

_settings = Settings()

def _fresh_client():
    return httpx.Client(base_url=_settings.API_URL, timeout=30)

def _register_and_login():
    """Helper: register a fresh user and return (email, password, tokens)."""
    client = _fresh_client()
    email = unique_email()
    password = "TestPass123!"
    client.post("/auth/register/", json={
        "email": email, "password": password, "password_confirm": password,
        "first_name": "Test", "last_name": "User"
    })
    login_resp = client.post("/auth/login/", json={"email": email, "password": password})
    client.close()
    return email, password, login_resp.json()
```
